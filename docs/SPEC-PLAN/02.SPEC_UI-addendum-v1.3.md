## A. Lưu đồ thao tác UI (owner-centric)

`emailOwner` là **context toàn cục** chọn ở Topbar. Mọi trang đọc `currentOwnerId` từ store. Chọn owner → chọn nghiệp vụ → 6 nhánh chức năng → hội tụ về Extracted Data + Variables (vòng lặp trích xuất/tái dùng).

**Quy tắc bố trí:**
- Owner selector ở Topbar là nguồn context toàn cục; đổi owner → mọi trang đang mở cập nhật ngay, không reload.
- Thứ tự sidebar theo trục owner: **Owners → Credentials → Fetch Builder → Extracted Data → Variables → Services & Resources**.
- Mỗi trang khi chưa chọn owner → empty state có CTA "Chọn owner" (không màn trắng/lỗi).

## B. Bộ UI Test theo luồng

> Cách dùng: sau mỗi thay đổi code, chạy các suite dưới đây theo đúng thao tác, tick từng bước. Hết xanh = luồng còn chuẩn. Mỗi bước ghi rõ **thao tác** và **kỳ vọng**.

### Golden rules — kiểm mỗi lần (vi phạm = chưa đạt dù bước khác xanh)
- Không có `alert/confirm/prompt` của browser (mọi thứ qua modal `ui.notify`/`ui.confirm`).
- Modal có nút ✕, **click ngoài KHÔNG đóng**, ESC đóng, tự scrollbar.
- Mọi button có **icon + tooltip** (mô tả chức năng + kết quả).
- Mọi màu qua CSS variable trong `tokens.css` (không hardcode).
- Reveal credential luôn qua **confirm** + ghi audit log.

### TC-1 · Owner làm trục context (Nhóm 2)
| # | Thao tác | Kỳ vọng |
| - | -------- | ------- |
| 1 | Mở app, nhìn Topbar | Có owner selector rõ + theme toggle bên cạnh |
| 2 | Chưa chọn owner, mở 1 trang bất kỳ | Empty state có CTA "Chọn owner", không màn trắng |
| 3 | Chọn 1 owner ở Topbar | Trang load dữ liệu đúng owner đó |
| 4 | Đổi sang owner khác | Mọi trang đang mở cập nhật context ngay, không reload |
| 5 | Kiểm thứ tự sidebar | Owners → Credentials → Fetch Builder → Extracted Data → Variables → Services & Resources |

### TC-2 · Quản lý Owner + Credentials (3.1)
| # | Thao tác | Kỳ vọng |
| - | -------- | ------- |
| 1 | Vào trang Owners | Bảng/card: email · isSaveRtdbEmail · createdAt · số credential |
| 2 | Tạo owner mới qua modal | Modal có ✕, click ngoài không đóng; lưu xong notify success |
| 3 | Click 1 owner | Panel chi tiết: credentials ở dạng masked |
| 4 | Bấm reveal credential | Confirm modal trước khi lộ; ghi audit log |
| 5 | Sửa rồi Xóa owner | Xóa có confirm cảnh báo kèm credentials; danh sách cập nhật |

### TC-3 · Fetch Builder → Extract → Execute (3.2, 3.3)
| # | Thao tác | Kỳ vọng |
| - | -------- | ------- |
| 1 | Từ màn owner bấm "Tạo Fetch Builder" | Builder mở với owner tiền điền sẵn |
| 2 | Paste curl | Parse ra method/url/headers/body; token map thành {{credential}} |
| 3 | Thêm/sắp xếp step, mở step editor | Có method · url · headers · body · extract (field + JSONPath) |
| 4 | Bấm "Test API" trên step | Gọi thật, response format đẹp để chọn JSONPath; có Copy curl |
| 5 | Lưu thành mẫu (preset) | Mẫu vào node flow-presets, dùng lại cho owner khác |
| 6 | Execute flow | Hỏi input runtime qua modal → stepper → lưu history + logs |

### TC-4 · Extracted Data & Variables (3.2, 3.4)
| # | Thao tác | Kỳ vọng |
| - | -------- | ------- |
| 1 | Sau execute, vào Extracted Data | Bảng: field · value · template nguồn · thời điểm; lọc theo owner |
| 2 | Sửa / Xóa 1 giá trị trích | CRUD đầy đủ; xóa qua confirm |

| 4 | Vào Variables, kiểm 2 tab | Có tab Global và Theo owner; CRUD qua modal |

### TC-5 · Services & Resources động (Nhóm 4)
| # | Thao tác | Kỳ vọng |
| - | -------- | ------- |
| 1 | Vào nav Services & Resources | Danh sách service động; host là combobox |

| 3 | Thêm resource type + gắn template | Vd repo/workflow/secret, mỗi cái gắn 1 template API |

| 5 | Thử action ghi (tạo/xóa secret) | Hành động ghi có confirm trước khi chạy |
